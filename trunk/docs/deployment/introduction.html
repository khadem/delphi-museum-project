Deploying Delphi: Introduction

This is a draft page, in ongoing development. It is still rough, but we are exposing it to the community for review and to help anyone considering a Delphi deployment. We are actively updating all of the documentation, so it is best to use this resource online, rather than printing out a version.

This is an overview of the deployment of Delphi, an open source software system that helps open up access via the Web to collections of artifacts held by museums, historical societies, archeological repositories and similar heritage institutions.

Delphi uses artificial intelligence and natural language processing to make it more effective and enjoyable to find individual artifacts in collections, and to uncover relationships between artifacts, than is possible through traditional searching and browsing.

To learn more about Delphi, see Delphi: Bibliography (TBD).

This deployment overview serves two purposes:

  • To document the current prototype instance of Delphi, in production use at the Phoebe A. Hearst Museum of Anthropology (PAHMA) at the University of California, Berkeley.

  • To make it easier to deploy and run your own instance of Delphi, at an institution other than PAHMA.

Deployment components

Delphi has three major components:

  1. The Delphi "collections browser," a web-based application that allows virtual museum visitors to explore the museum's collections over the Internet.  Visitors can browse and search the collections using concepts such as culture and materials, as well as using traditional search terms.
  2. A data repository is accessed by the collections browser.  This repository stores the data about the artifacts in the collections ("metadata"), as well as a semantic index containing the associations between each artifact and the set of concepts that pertain to it.

  3. A set of back-end tools build and maintain the data in the data repository.  Thereafter, that data is accessed directly by the collections browser.

Collections browser

The Delphi collections browser is supported by PHP libraries and PHP-driven content and template files.  It also accesses data in a repository - a MySQL database - whose contents are maintained using a set of back-end tools.

To learn more about the collections browser and its sub-components, see Deploying Delphi: Components.  (Change this to Deploying Delphi: Collections browser.)

Back-end tools

A Java application provides support for text analysis, semantic indexing, and related support to build and maintain the database that supports the online browser.  A 'sed' script is also used to perform some initial data cleanup, required before this set of processes can take place.

To learn more about these back-end tools, see Deploying Delphi: Back-end Tools.
