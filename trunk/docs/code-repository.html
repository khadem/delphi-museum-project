Delphi: Code Repository

This document describes the code repository used to manage the source code, configuration files, and other artifacts of the Delphi project, used by the Phoebe A. Hearst Museum of Anthropology (PAHMA) at the University of California, Berkeley.

Google Code Repository

The Delphi project currently uses a free, open source code hosting site on Google Code.  The site is located at:

http://code.google.com/p/delphi-museum-project/

(This site can also be accessed via an alternate URL, http://delphi-museum-project.googlecode.com/)

This site was chosen, in part, because it offered a good issue tracking system.  Project members had previously experimented with another open source code hosting service, SourceForge, and there is an obsolete Delphi project site on that service as well.  (Wikipedia also offers an article comparing multiple services of this type.)  If and when UC Berkeley's Information Services and Technology (IST) offers a comparable hosting site in-house, this could be considered for hosting the service in the future.

Site administration and permissions

The Delphi project site on Google Code is currently owned, and hence administered, by Patrick Schmitz.  The site owner(s) can add or delete owners and members via an "Administer" tab on the site, accessible only to them.

There is a set of project members, any of whom can commit code, edit wiki pages, or otherwise make changes to the content of the site.

Finally, all other visitors to the site have broad, primarily read-only access: to browse or download source code, read and comment on wiki pages, and read documentation (such as this document).

All project owner(s) and members are identified by their Google accounts, rather than by email addresses provided by UC Berkeley or other institutions.

Version control repository

Google Code offers a Subversion-based version control repository.  A current version of nearly all of Delphi's code and related artifacts is maintained there.  The repository also offers access to older revisions of each of these files, and to notes about the changes made in each successive release.

(One notable exception are the back-end 'sed' scripts that are used for initial cleanup of the data provided by PAHMA.  As noted in Deploying Delphi: Back-end Tools,  these scripts contain some sensitive information, so they are not included in this publicly-accessible repository.)

You can browse Delphi's code and related artifacts can be browsed via the web by clicking the "Source" tab on the project's Google Code site, then the "Browse" link under that tab.  More typically, however, you'll access this code base via a Subversion client - a piece of software that allows one to interact with Subversion repositories.  The Subversion website maintains a list of these clients (http://subversion.tigris.org/links.html#clients), and a Wikipedia article offers a comparison matrix of many popular clients (http://en.wikipedia.org/wiki/Comparison_of_Subversion_clients).

Public access to Delphi's code

Anyone can use a Subversion client to download Delphi's code.  The public URL to which anyone can point a Subversion client to download the project's code and related artifacts is:
http://delphi-museum-project.googlecode.com/svn/trunk/ delphi-museum-project-read-only
From a command-line version of the Subversion client, the command to check out the project to a local directory named "delphi-museum-project-read-only" is:
svn checkout http://delphi-museum-project.googlecode.com/svn/trunk/ delphi-museum-project-read-only

Private (project member-only) access to Delphi's code

Project members can also use a Subversion client, not only to download code but also to make changes within the project repository.  The private URL to which they can point a Subversion client is available by clicking the "Source" tab on the project site, after they have logged into their Google accounts.

Project members use their Google account as a username, and a googlecode.com password.  They can generate or update this password by clicking the "Source" tab on the project site, and following the link provided on the "Checkout" page.  (This link only appears for logged-in project members.)
