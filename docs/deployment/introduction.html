Deploying Delphi: Introduction

This is a draft page, in ongoing development. It is still rough, but we are exposing it to the community for review and to help anyone considering a Delphi deployment. We are actively updating all of the documentation, so it is best to use this resource online, rather than printing out a version.

This is an introduction to and overview of the deployment of Delphi, an open source software system that helps open up access via the Web to collections of artifacts held by museums, historical societies, archeological repositories and similar heritage institutions.

Delphi uses artificial intelligence and natural language processing to make it more effective and enjoyable to find individual artifacts in collections, and to uncover relationships between artifacts, than is possible through traditional searching and browsing.

To learn more about Delphi, see Delphi - Bibliography (TBD).

This deployment overview serves two purposes:

  • To document the current prototype instance of Delphi, in production use at the Phoebe A. Hearst Museum of Anthropology (PAHMA) at the University of California, Berkeley.

  • To make it easier to deploy and run your own instance of Delphi, at an institution other than PAHMA.

Deployment components

There are three major components to the current Delphi deployment used by the Phoebe A. Hearst Museum of Anthropology (PAHMA) at the University of California, Berkeley.  These components, along with their associated sub-components, will also be present in any highly similar instances of Delphi that are deployed and run outside PAHMA:

  • The Delphi "collections browser," a web-based application, allows virtual museum visitors to explore the museum's collections over the Internet.

  • A data repository stores the data that is accessed by the collections browser.

    This includes     data about the artifacts in the collections ("metadata"), plus a "semantic index" containing the associations between each artifact and the set of concepts that pertain to it.

  • A set of back-end tools process the data obtained from the museum, and load the data into the data repository.
More detail about each of these three major components follows:

Collections browser

The Delphi "collections browser," a web-based application, allows virtual museum visitors to explore the museum's collections over the web.  Visitors can search for artifacts, via both concepts and keywords, and can interactively browse their various "facets" (e.g. location, culture, materials).

The collections browser currently runs in the Apache Web Server and at its core consists of PHP-driven content and template files, supported by PHP libraries; JavaScript code and libraries; CSS style sheets, and a Flash-based tool to allow users to zoom-and-pan through images.

To learn more about the collections, see Deploying Delphi: Collections Browser.

Data repository

The data repository contains the data that is accessed by the collections browser.  This data is also sometimes used for reporting and other purposes.

Currently, the data repository is a MySQL relational database.  Delphi can also potentially use other relational database software, or in the future, a "triple store" or other type of database-like repository.

The data repository stores two types of data:

  • Data about artifacts in the collections ("metadata").

    Purposes of this data:  This data is displayed when the Delphi collections browser provides details about individual artifacts.

    In addition, this data is "mined," via natural language processing, to help build a set of concepts that more fully describe the collection and then to associate those concepts with each of the individual artifacts in the collections.

    Source of this data:  In the instance of Delphi used by PAHMA, this metadata comes directly from that museum.  Future instances of Delphi at other sites will obtain this data from their own museums or heritage institutions.  The format of this data will vary, depending on how the museum or heritage institution stores the met about its collections.

  • Data about the concepts associated with these artifacts (a "semantic index").

    Purpose of this data:  This data helps users of Delphi's collections browser locate artifacts by concepts, and help uncover relationships between artifacts.

    Source of this data: This data is generated by Delphi's back-end tools (below), based on the data provided by the museum.    Future instances of Delphi at other sites will build their own ontologies.
To learn more about the data repository, see Deploying Delphi: Data Repository.

Back-end tools

A set of back-end tools are used to take the data received from PAHMA, process that data, and load it into the data repository.

There are currently two back-end tools:

  • A 'sed' script for initial cleanup of the data.

  • A Java-based, GUI application for further processing of the cleaned data.

    This application provides support for text analysis, semantic indexing, and related support to build and maintain the database that supports the online browser.  Delphi's administrators and others to analyze and process data about the artifacts in the collection - both metadata and semantic index data - and to load this data into the data repository.
